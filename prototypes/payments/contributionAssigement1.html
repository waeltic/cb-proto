Club settings – Billing model

Damit USA/andere Clubs beim heutigen Modell bleiben, wird pro Club ein Billing Model gewählt.

Legacy: wie heute (Products→Orders→Invoices). DE: Assignments erzeugen periodische Invoices.

Wird für service_mode = club_year verwendet (Ende = Start - 1 Tag im Folgejahr).

🏛️
DE-Regeln (V1+)
  • service_mode = club_year oder open
  • billing_interval = once/monthly/quarterly/yearly
  • Kein Pro-Rata, keine Teilperioden, Enddatum wird gerundet
  • Invoice zeigt immer period_start / period_end
  • Optional auto_renew (Default create_new)

What changes in DE mode?

Nur die Quelle wird erweitert – Invoices/Transactions bleiben wie heute.

Add
  • ContributionAssignment (Beitragszuweisung)
  • Invoice.period_start / period_end
  • Link invoice/orderItem → assignment
  • Neue UI: Assignments & Contribution overview
Keep
  • OrderItem snapshotting (price/tax/discount)
  • Invoices & Transactions + SEPA
  • Nightly/Batch charging pipeline
  • Manual invoice flows

Service-Rahmen (DE)

club_year nutzt Club-Vereinsjahr Startdatum. open hat keine übergeordnete Serviceperiode.

Club year used:
Start: 01.01 → End: 31.12 (from club settings)
Beispiel: Eintritt 15.03.2026 → Service: 15.03.2026 – 31.12.2026
